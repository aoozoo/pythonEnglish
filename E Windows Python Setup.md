# Installing Python On a PC



本节课专门针对 Windows (PC) 用户讲解 Python 的安装流程。如果你使用的是 Mac 或 Linux 系统，请直接跳过本节，因为系统底层差异会导致教程信息冲突。

相比于在 Windows 上配置 Ruby 等其他语言那令人头疼的过程，Python 的安装其实非常简单，几乎只需点击几下鼠标。但为了能顺利执行代码，我们必须掌握环境配置中最核心的一步，以及如何验证安装结果。

---

## 极简安装流程与核心陷阱
* **下载正确版本**：前往官方网站 `python.org`，进入下载页面（Downloads），选择为 Windows 下载，并且一定要认准 **Python 3.x** 版本（千万不要选老旧的 2.7 版本）。
* **生死攸关的勾选项 (Add to PATH)**：在运行安装程序时，请务必停下来，**一定要勾选界面底部的“Add Python to path”复选框**。
  * *DevOps 视角补充*：这个操作会自动将 Python 的执行路径写入 Windows 的全局环境变量（Environment Variables）中。如果不勾选，你的 PowerShell 将永远无法识别 `python` 这个命令。
* **重启终端环境**：安装完成后，如果你之前一直开着 PowerShell，必须将它关闭并重新打开。这是因为旧的终端进程无法动态感知刚刚注入的全新环境变量，重启即可让其生效。

## 验证安装：初探 REPL 环境
* **执行验证**：在全新的 PowerShell 窗口中，直接输入 `python` 并回车。如果一切顺利，你会看到输出版本号（通常是 3.6.4 或更高版本），这意味着安装成功。如果看到满屏的红色报错（提示术语未被识别），说明大概率是忘记勾选了 PATH 选项。
* **什么是 REPL？** 当你输入 `python` 并回车后，命令提示符会发生改变，你正式进入了 Python 的交互式解释器，也被称为 **REPL**。
  * **R (Read)**：读取你输入的代码。
  * **E (Evaluate)**：评估/计算代码逻辑。
  * **P (Print)**：打印出执行结果。
  * **L (Loop)**：循环等待你的下一次输入。
* **REPL 的正确使用场景**：这是一个绝佳的“代码试验田”。比如你输入 `1 + 1`，它会立刻输出 `2`。但我们不会在这里编写复杂的应用程序或分析 CSV 数据，日常开发还是要在独立的代码文件中进行。

## 工程师必备：如何安全退出？
进入了 REPL，该怎么回到普通的 PowerShell 呢？
* **常规指令**：输入 `quit()` 并回车（注意：必须带上括号）。
* **快捷键操作**：使用组合键 `Ctrl + Z`，然后敲击回车。
退出后，你可以随时再次输入 `python` 重新进入这个交互式环境。

## 疑难解答 (Troubleshooting)
* **不要慌**：如果在安装或执行过程中遇到问题，通常是因为你系统的具体 Windows 版本差异，或者是你以前无意中安装过其他版本的 Python 导致的冲突。
* **高效求助**：在课程讨论区提问时，请务必直接复制并粘贴你看到的完整错误日志（Error messages），这样更有利于快速定位并解决问题。



# Introducing Sublime Text

本节课讲师正式介绍了他将在课程中使用的代码编辑器：**Sublime Text**。作为初学者，拥有一个好用的编辑器能极大提升编写 Python 代码的幸福感。讲师演示了它的下载安装过程、基本的文件操作，以及初学者最容易踩坑的“语法高亮”触发机制。

---

## 为什么选择 Sublime Text？
* **免费且强大**：市面上有些 IDE（集成开发环境）在 30 天试用期后会向你收取高达一两百美元的费用。而 Sublime Text 提供了极其宽松的免费评估版本，且功能同样强大，非常适合初学者和预算有限的开发者。
* **语言无关性与流行度**：它不仅支持 Python，还支持 JavaScript 等几乎所有主流编程语言，在开发者社区中拥有极高的声誉和庞大的用户群。

## 安装与极简主义体验
* **一键安装**：只需访问 Sublime Text 官网，网站会自动识别你是 PC 用户，点击“Download for Windows”并一路“下一步”即可完成向导安装。
* **纯粹的代码环境**：与 Microsoft Word 这种富文本处理器不同，代码编辑器里没有插入图片、剪贴画（Clip art）等花哨功能。它剥离了一切干扰，专注于纯文本（Plain text）代码的编写。

## 核心操作与“语法高亮”避坑指南
作为 DevOps 工程师，我们最看重工具的效率，掌握以下快捷键和底层逻辑至关重要：
* **新建与保存**：使用快捷键 `Ctrl + N` 新建文件，`Ctrl + S` 保存文件。
* **文件后缀名的魔法**：这是新手最常问的问题。在编写 Python 代码时，**必须将文件命名为以 `.py` 结尾（例如 `demo.py`）**。
* **触发语法高亮 (Syntax Highlighting)**：如果你只是新建一个空白文件直接敲代码，文本会是单调的黑白色。只有当你将文件保存为 `.py` 后，Sublime 才会知道这是一个 Python 文件，从而激活紫色的数字、红色的运算符等友好的语法高亮和代码自动补全功能。**记住铁律：先保存并命名后缀，再写代码**。

## 个性化与项目级工作流
* **更换皮肤**：程序员都喜欢暗黑模式或酷炫的配色。你可以通过顶部菜单栏的 `Preferences -> Color Scheme` 来切换你看着最舒服的代码配色方案。
* **打开整个文件夹 (Open Folder)**：在后续的工程化实践中，我们的项目往往包含多个 Python 脚本 and 外部数据文件（如 CSV）。此时不要逐个打开文件，而是使用 `File -> Open Folder`（或直接将文件夹拖拽进软件），将整个项目目录挂载到编辑器侧边栏。这不仅方便文件切换，也是真实工业界开发的标准工作流。
* **高阶定制**：Sublime 拥有极高的可定制性，你可以编写自定义脚本、下载海量第三方插件。但为了不偏离主线，本课程不会深挖编辑器的高级玩法，而是将重心拉回 Python 语言本身。



# Your First Python Program

本节课我们迎来了激动人心的时刻——编写并执行第一行真正的 Python 代码。讲师演示了如何在 Sublime Text 中创建文件、编写代码，并在 PowerShell 中将其运行。核心知识点是 `print` 函数的使用，以及打通“编写-保存-执行”的完整工作流。

---

## 打通全链路：创建与执行
* **新建并规范命名**：在 Sublime 中使用 `Ctrl + N` 新建文件，并将其保存在桌面（Desktop）上。请将其命名为 `first_program.py`，`.py` 后缀是触发 Python 语法高亮和正确执行的关键。
* **终端导航跟进**：打开 PowerShell，利用上一章学到的导航技能，输入 `cd desktop` 切换到桌面目录，并用 `ls` 命令确认你的文件确实安详地躺在那里。
* **第一次执行**：在终端输入 `python first_program.py`（记得善用 Tab 键补全）并敲击回车。

## 黑暗中的低语：为什么没有输出？
* **盲写代码的陷阱**：如果你在文件里只写了一个带引号的 `"hello"`，保存后去终端执行，你会发现终端毫无反应，什么都没发生。
* **静默即安全 (Silent is Gold)**：在 Linux/Unix 哲学中，没有消息就是好消息。不报错意味着 Python 成功解析并运行了你的代码。
* **数据迷失**：虽然代码跑通了，但由于你没有给系统下达“输出”的指令，这段数据就像是“向黑暗中低语”，声音直接消失在 Python 的无尽深渊中。

## 核心语法：Print 函数
* **召唤标准输出**：为了让程序开口说话，我们需要引入第一个 Python 内置功能——`print` 函数。
* **标准语法**：输入 `print`，紧跟一对英文括号 `()`，不要有空格，然后将你想输出的文字放入括号内。
* **DevOps 的肌肉记忆：保存！**：修改代码后，**千万记得按 `Ctrl + S` 保存文件**。这是无数新手（甚至老手）最容易翻车的地方——跑了半天发现结果不对，原来是代码没保存。
* **见证奇迹**：回到终端，按方向键 `↑` 调出上一条执行命令，再次回车，你就能在 PowerShell（即标准输出 stdout）中看到打印出的 hello 了。
* **跨平台一致性**：Python 是跨平台的。如果你把这个 `.py` 文件发给用 Mac 的同事，他们在 Mac 终端里运行，会得到完全一致的输出结果。

## 随堂练习预告
* **熟悉测试平台**：接下来，Udemy 平台会提供一个极简的练习题，要求你打印自己的名字。
* **目的不在于考核**：这个练习并非为了刁难你，而是让你跑通平台的提交流程：在网页里写代码 -> 点击提交 -> 观察后台自动化测试的反馈。



# Important Note on Print and the Python Interpreter

本节课讲师补充解答了一个初学者极易混淆的核心问题：在 Python 交互式解释器中直接输入数据，与在独立脚本中使用 `print` 函数到底有何区别。核心结论是：解释器会自动打印所有执行结果，但在实际编写脚本时，计算会在后台默默完成，你必须显式调用 `print` 才能在屏幕上看到输出。

## 解释器 (REPL) 与脚本 (Script) 的根本差异
* **临时试验田 vs 生产环境**：Python 解释器（即终端直接输入 Python 运行的环境）主要用于快速测试或调试代码，真正的应用程序绝不会在这里编写。
* **REPL 的底层逻辑**：REPL 代表读取 (Read)、评估 (Evaluate)、打印 (Print)、循环 (Loop)。当你输入像 `43 + 1` 这样的算式时，系统会读取代码，评估计算出 44，自动打印在屏幕上，然后循环等待下一次输入。
* **自动输出的假象**：正是因为 REPL 带有自动打印的特性，在里面你即使不敲 `print` 也能看到结果，这会给初学者带来认知误区。

## 脚本执行：沉默是金
* **无声的计算**：如果你把同样的 `43 + 1` 写进一个 Python 脚本文件（如 `demo.py`）并执行它，终端里看起来就像什么都没发生一样。
* **后台默默打工**：此时程序其实已经被 Python 成功执行并计算出了结果，但由于没有任何显式的输出指令，这个结果永远不会展示给你，直接就消失了。

## Print 的真正使命
* **强制输出的指令**：要在运行脚本时看到结果，你必须用 `print()` 将内容包裹起来。这样就是明确要求系统将结果输出到终端屏幕上。
* **SRE 的核心直觉**：在解释器里，系统会主动给你反馈；但只要是运行独立的 Python 脚本，想要输出任何信息，你必须自己手写 `print`。
