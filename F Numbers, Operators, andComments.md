# Numbers: Ints and Floats

本节课我们正式开始接触 Python 处理数据的方式，首当其冲的就是数字。在现实世界中，数字是不分类的，但在 Python 的底层世界里，为了极致的内存优化，数字被严格划分了类型。讲师重点讲解了整数（Integers）和浮点数（Floats）的区别、如何使用内置函数侦测数据类型，以及混合运算时的数据类型自动转换规则。

---

## 为什么数字需要分类？(内存与空间的权衡)
* **现实与代码的鸿沟**：在现实生活中，1、-20 和 57.775 统统都被称为“数字”。但在 Python 中，它们在底层是被区别对待、分开存储的。
* **底层二极管的逻辑**：我们在代码里写的所有数据，最终都会被转换为 0 和 1 存储在计算机内存中。
* **无限与有限的较量**：讲师给出了一个非常精彩的极客视角。对于整数（Whole numbers），0 和 1 之间没有其他整数，它们是确定的、有限的跳跃。但是对于小数而言，0 和 1 之间存在着无限个可能的数字。
* **空间分配差异**：正因为小数具有无限的精度可能性，Python 必须在内存中为它们预留和消耗更多的空间。因此，将数字区分为整数和带小数点的浮点数，本质上是为了节约内存。

## 核心概念：整数 (Ints) 与 浮点数 (Floats)
* **四大门派**：Python 内部其实有四种数字数据类型（包括复数 Complex 和长整型 Long），但作为开发者，绝大多数时候你完全不需要理会那两种冷门类型。我们需要死磕的只有两个：整数和浮点数。
* **整数 (Integers / Ints)**：也就是完整的自然数，包括正数和负数（例如 4、57、-3000）。它们占用较少的内存空间。
* **浮点数 (Floating Point / Floats)**：带有小数点的数字，同样包含正负数（例如 6.11、-1.333）。它们占用更多的内存空间。
* **一个反直觉的陷阱**：在现实中 1 和 1.0 是一样的，但在 Python 中，只要带了小数点和零（即 `1.0`），系统就会强制将其作为“浮点数 (Float)”来处理。

## 极客神技：使用 type() 函数侦测数据类型
* **验明正身**：在 Python 交互式解释器（REPL）中，讲师提前演示了一个非常好用的工具：`type()` 函数。
* **实操演示**：
  * 当你输入 `type(9)` 时，Python 会返回 `<class 'int'>`，告诉你这是一个整数。
  * 当你输入 `type(9.0)` 时，Python 会返回 `<class 'float'>`，告诉你这是一个浮点数。
* 这个函数在未来我们做数据清洗和排查 Bug 时会极其常用。

## 算术运算与“浮点数传染”定律
* **强大的计算器**：Python 可以轻松处理各种加减乘除运算，比如 `1 + 1`、`5 - 3`，甚至组合运算 `1 + 4 - 2`。
* **数据类型融合**：当你在运算中混合使用了整数和浮点数（例如 `1 + 1.0` 或者 `20 + 0.777`）时，会发生什么？
* **底层规则**：Python 看到算式中存在浮点数，就会立刻做出妥协，将最终的计算结果也转化为浮点数。因此 `1 + 1.0` 的结果不是 2，而是 `2.0`；`20 + 0.777` 的结果是 `20.777`。**一句话总结：只要有浮点数参与运算，返回值必定是浮点数。**



# Numbers: Basic Math

本节课我们将学习 Python 中的基础数学运算符。虽然加减乘除看起来和我们在现实生活中用的一模一样，但 Python 的除法运算隐藏着一个非常关键的“怪癖”。此外，讲师还带我们复习了如何利用括号来控制复杂算式的运算优先级。

## 核心基础运算符
* **常见符号**：加法 (`+`)、减法 (`-`)、乘法 (`*`) 和除法 (`/`)。
* **混合运算**：正如上一节课所说，如果你在加减乘法中混合了整数和浮点数，比如 `40 - 4.5`，因为表达式中包含了浮点数，其结果必然也是一个浮点数。

## 除法的“怪癖”：永远返回浮点数
* **反直觉的机制**：当我们用一个整数去除以另一个整数时，比如 `1 / 2`，结果会是什么数据类型？
* **Python 的选择**：在某些其他的编程语言中，整数相除会向下取整（直接丢弃小数部分返回 0）。但在 Python 中，**除法运算 (`/`) 永远都会返回一个浮点数 (Float)**。
* **实操验证**：在解释器中输入 `1 / 2`，即使两边都是整数，Python 返回的也是浮点数（例如 `0.5`）。输入 `1 / 3` 则会返回 `0.3333...` 这样的浮点数。
* **预告**：如果你确实需要在除法中忽略小数并得到一个整数，Python 提供了另一种特殊的运算符，讲师将在下节课专门讲解。

## 运算优先级 (PEMDAS)
* **经典数学法则**：Python 完全遵循我们在现实世界中使用的标准数学运算优先级。在英文中通常用缩写 **PEMDAS** 来辅助记忆：括号 (Parentheses)、指数 (Exponents)、乘 (Multiplication)、除 (Division)、加 (Addition)、减 (Subtraction)。
* **执行顺序演示**：对于算式 `1 + 2 * 5 / 3`，Python 会严格按照优先级，先算乘法，接着算除法，最后再算加法。
* **括号的绝对统治力**：和用纸笔算数学题一样，如果你想打破默认的优先级，只需加上括号即可。比如改成 `(1 + 2) * 5 / 3`，Python 就会优先计算括号里的加法。这也是我们在编写复杂公式时，确保逻辑准确且代码易读的最佳手段。



# Quick Note: Comments

这节课虽然插入在数学计算的章节中，但它讲解的是一个极其重要且通用的编程基础：代码注释。讲师演示了如何在 Python 脚本中添加注释，以及在团队协作中注释的真实意义。

---

## 注释的底层逻辑与语法
* **注释的本质**：注释就是我们在代码中写下的“小抄”或说明笔记，这些内容在程序运行时会被 Python 完全忽略，绝不会被实际执行。
* **魔法符号 `#`**：在 Python 中，通过输入 `#`（井号/哈希号，讲师提到它还有一个冷门名字叫 octothorpe）来开启一段注释。
* **“隐身斗篷”机制**：这个符号的作用是告诉 Python 解释器，直接无视 `#` 之后同一行内的所有内容。它就像是给文本披上了一件“隐形斗篷”。哪怕你把正常的、原本可以执行的 Python 代码放在 `#` 后面，它也只会被当成普通文本，不会有任何反应。

## IDE 极客提效神技
* **一键注释/解开**：如果你使用的是 Sublime Text 或 Atom 等现代化的代码编辑器，只需要选中目标行，然后按下 `Command + /`（Windows/Linux 用户为 `Ctrl + /`），编辑器就会自动在行首添加或移除 `#`，实现一键整行注释。

## SRE 视角：为什么必须写注释？
* **写给现在的自己（学习利器）**：对于初学者而言，单看代码有时候很难直观地理解它的作用，尤其是在早期阶段。通过添加注释，你可以随时提醒自己这段逻辑到底是在干什么。讲师极力推荐初学者多写注释，哪怕每行都写，只要能帮助理解就不算浪费时间。
* **写给同袍的交接文档（团队协作）**：在真实的工业环境里，你可能会在一个 10 人团队中工作，写下 20 行复杂的业务代码。如果你在顶部加一句注释（比如“这个函数用来处理 X 或 Y 逻辑”），其他同事扫一眼就能明白其核心功能，完全不需要逐行死磕你的代码去猜测意图。
* **架构文化的差异**：不同的科技公司有着截然不同的代码评审（Code Review）文化。有些团队强制要求详尽的注释；而另一些团队则推崇“代码即文档”的理念，认为真正优雅的代码应该是自解释的（self-explanatory），根本不需要额外的注释来画蛇添足。



# Numbers: Weirder Operators

本节课是数学章节的收官之作。讲师带我们接触了三个在日常数学中不太常见、但在编程世界里非常核心的特殊运算符：指数（`**`）、取模（`%`）和整除（`//`）。掌握它们能帮我们解决很多特定的工程逻辑需求。

---

## 指数运算 (Exponentiation: `**`)
* **幂运算**：与表示乘法的单星号 `*` 不同，双星号 `**` 用于将一个数字提升到指定的幂次方。例如，输入 `2 ** 3`，等同于 2 乘 2 乘 2，计算结果为 8。
* **极客用法（开根号）**：除了计算整数次幂，你还可以用它来求平方根或其他次方的根，只需要将指数设为小数或分数即可。例如，输入 `49 ** 0.5` 会计算 49 的平方根，并返回浮点数 `7.0`。

## 取模运算 (Modulo: `%`)
* **核心功能**：这可能是初学者最陌生、但在开发中极其高频的运算符。它又被称为“余数运算符（Remainder operator）”。它执行除法操作，但不返回商，而是只返回**余数**。
* **计算逻辑**：例如计算 `10 % 3` 时，3 最多能完整地放进 10 里面 3 次（3 * 3 = 9），剩下的余数就是 1。
* **SRE/开发常用场景**：取模运算最经典的应用场景就是**判断一个数字是奇数还是偶数**。任何数字只要对 2 取模（`num % 2`），如果它能被 2 整除（余数为 0，例如 16），就是偶数；如果余数为 1（例如 13），那就是奇数。

## 整除运算 (Integer Division: `//`)
* **对冲常规除法的“怪癖”**：上一节课我们提到，常规除法 `/` 无论如何都会返回一个浮点数。如果你在业务逻辑中坚决不想要浮点数，就可以使用双斜杠 `//` 进行整除运算。
* **强制向下取整**：它会执行除法，但会直接砍掉（向下取整）所有的小数部分，只返回一个干净的整数 (Int)。例如，输入 `6 // 7` 会直接返回 `0`。
